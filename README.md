針對 **`start-04-unit-touch-basics` (觸控事件的生命週期：從指尖到搖桿)** 單元，這是學員建立「觸覺操控感」的最底層開發課。它要求學員具備 **多點觸控辨識 (Multi-point Tracking)** 與 **座標映射 (Coordinate Mapping)** 的硬核心技術，學習如何處理 `touchstart`、`touchmove`、`touchend` 的生命週期，並解決行動裝置上臭名昭彰的「300 毫秒點擊延遲 (300ms Click Delay)」問題。

以下是在 **GitHub Classroom** 部署其作業 (Assignments) 的具體建議：

### 1. 範本倉庫 (Template Repo) 配置建議
觸控基礎作業的核心在於「事件的捕捉、追蹤與放開」，範本應包含觸控座標偵錯工具與多點辨識框架，建議包含：
*   **📂 `index.html`**：**觸控視覺化實驗室 (Touch Sandbox)**。預設包含一個「觸控日誌面板」，學員在開發時能即時看到每根手指的 `identifier` 以及 `clientX / clientY` 的動態座標，這對理解多點觸控至關重要。
*   **📂 `touch.js`**：**座標映射實作區**。學員在此實作 `Map` 或物件來儲存多根手指的狀態資訊。預留 `// TODO` 引導學員從 `touches` 或 `changedTouches` 中提取資料並進行座標轉換。
*   **📂 `README.md`**：**觸控反應審計表 (Touch Response Audit)**。提供模板要求學員紀錄：使用 `click` 與 `touchstart` 發送指令，在硬體端的反應速度感知差異為何？三指同時操作時，系統是否能正確分辨誰是誰？
*   **📂 `assets/`**：**座標系說明圖**。標示瀏覽器 `Client` 座標與網頁 `Page` 座標的差異圖示，輔助學員進行座標映射。

### 2. 作業任務部署細節

#### 任務 1：「零延遲」按鈕饋送實作 (Zero-Latency Button Lab)
*   **目標**：徹底消除網頁的「遲鈍感」。學習利用 `touchstart` 取代 `click` 事件以繞過瀏覽器預設延遲，並處理「幽靈點擊 (Ghost Clicks)」。
*   **Classroom 部署建議**：
    *   **即時回饋實作**：在觸碰瞬間立即改變按鈕 CSS 樣式（發光或縮放），讓使用者感到「指尖即響應」。
    *   **事件攔截挑戰**：正確使用 `e.preventDefault()` 阻止瀏覽器在觸碰後 300ms 補發一個多餘的 `click` 事件。
    *   **驗證要點**：**延遲感審核**。學員的操作是否達到了「按下即觸發」的體感？是否妥善處理了 `touchend` 的收尾邏輯（放開即停止）？

#### 任務 2：多點觸控追蹤與座標映射 (Multi-Touch Mapping Lab)
*   **目標**：掌握「點對點」的動態追蹤。學習利用 `identifier` 屬性辨識不同手指，確保左手控制搖桿、右手控制按鈕時互不干擾。
*   **Classroom 部署建議**：
    *   **識別碼管理實作**：使用 `Map` 結構記錄每根手指的流水號，並在 `touchmove` 中只更新 ID 對應的物件座標。
    *   **邊界條件測試**：實作當手指滑出控制區範圍時，如何正確捕捉到 `touchend` 或 `touchcancel` 以安全停止馬達。
    *   **驗證要點**：**追蹤準確性審核**。多指同時操作下，搖桿是否會因為另一根手指的加入而發生「瞬間位移」？座標轉換公式是否正確考慮了控制區的選取範圍？

#### 任務 3：全方位介面安全鎖 (Touch Interface Safety Lab)
*   **目標**：打造「鋼鐵般」的穩定介面。防止雙指縮放或雙擊放大導致控制介面崩潰，讓 Web 表現得像原生 App。
*   **Classroom 部署建議**：
    *   **縮放封鎖實作**：應用 CSS `touch-action: none` 與 JS 事件攔截，徹底禁止瀏覽器預設的捏合縮放手勢。
    *   **動態回饋機制**：為長按動作加入視覺化進度條或發光脈衝，提示使用者操作正在進行中。
    *   **驗證要點**：**介面穩固度審核**。在瘋狂操作下，頁面是否始終維持固定比例而不縮放、不偏移？是否正確設定了 `{ passive: false }` 以確保攔截生效？

### 3. 多點辨識與座標健壯性點評標準 (Tactile Engineering Standards)
此單元的價值在於 **「對多個並行狀態的識別能力與對座標系轉換的幾何處理能力」**：
*   [ ] **延遲消除度**：是否徹底根除了傳統點擊事件帶來的體感延遲？
*   [ ] **多指穩定度**：代碼是否能優雅地處理多根手指同時存在、分別離場的複雜狀態？
*   [ ] **座標映射精準度**：手指在螢幕上的移動量是否能線性且正確地映射到機器人的動力輸出值上？
